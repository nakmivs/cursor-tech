# 企业风险报告撰写指南

## 概述

本 skill 指导如何撰写专业的企业风险分析报告，包括数据检索、风险识别、分析评估和报告生成。

## 数据检索流程

### 步骤 1：使用 Mitaso MCP 检索数据

使用 `user-mitaso` MCP 服务器的三个工具检索信息：

**1. metaso_web_search - 搜索相关资料**

```
工具: metaso_web_search
参数:
  - q: 搜索关键词（如"[企业名称] 财务风险"、"[行业] 政策法规变化"）
  - size: 返回数量（建议 15-20）
  - scope: 搜索范围
    - "webpage" - 网页新闻
    - "document" - 研究报告
    - "paper" - 学术论文
  - includeSummary: true（包含摘要）
```

**建议搜索关键词组合：**
- `[企业名称] 财务报表 年报`
- `[企业名称] 诉讼 处罚 风险`
- `[行业名称] 政策法规 监管`
- `[企业名称] 竞争对手 市场份额`
- `[行业名称] 发展趋势 风险因素`

**2. metaso_chat - 智能问答获取分析**

```
工具: metaso_chat
参数:
  - message: 具体问题（如"分析[企业名称]的主要财务风险"）
  - model: "fast"（快速响应）或默认
```

**3. metaso_web_reader - 读取详细内容**

```
工具: metaso_web_reader
参数:
  - url: 目标网页 URL
  - format: "markdown"（推荐）或 "json"
```

### 步骤 2：信息整理

将检索到的信息按以下类别整理：
1. 企业基本信息（规模、主营业务、股权结构）
2. 财务数据（营收、利润、负债率、现金流）
3. 行业环境（市场规模、竞争格局、发展趋势）
4. 监管政策（相关法规、政策变化）
5. 风险事件（诉讼、处罚、负面新闻）

## 报告撰写规则

### 结构规范

报告必须包含以下章节：

1. **执行摘要**（200-300字）
   - 企业概况一句话描述
   - 核心风险点（不超过3个）
   - 整体风险评级
   - 关键建议

2. **企业概况**
   - 基本信息（名称、成立时间、注册资本）
   - 主营业务与商业模式
   - 股权结构与管理层
   - 行业地位

3. **风险分析**（核心章节）
   - 财务风险
   - 运营风险
   - 合规风险
   - 战略风险
   - 外部环境风险

4. **风险评估矩阵**
   - 风险等级划分（高/中/低）
   - 发生概率评估
   - 影响程度评估

5. **建议与对策**
   - 针对性的风险缓解措施
   - 优先级排序

6. **数据来源与局限性**
   - 引用来源列表
   - 分析局限性说明

### 风险等级标准

| 等级 | 标识 | 判定标准 |
|------|------|----------|
| 高风险 | 🔴 | 可能导致重大财务损失或业务中断 |
| 中风险 | 🟡 | 需要关注，有一定负面影响 |
| 低风险 | 🟢 | 影响有限，可接受范围内 |

### 写作风格

- 使用客观、专业的语言
- 数据和结论需有来源支撑
- 避免主观臆断，如需推测应明确标注
- 重要结论使用加粗或列表突出
- 数字精确到合理位数

### 引用规范

每个关键数据或结论后标注来源：

```
公司2024年营收达到50亿元，同比增长15%。[来源：公司年报]
```

## 工作流程检查清单

执行报告撰写时，复制此清单追踪进度：

```
报告撰写进度：
- [ ] 1. 确认目标企业和分析范围
- [ ] 2. 使用 metaso_web_search 检索企业信息
- [ ] 3. 使用 metaso_web_search 检索行业政策
- [ ] 4. 使用 metaso_chat 获取风险分析见解
- [ ] 5. 使用 metaso_web_reader 读取关键资料详情
- [ ] 6. 整理信息并分类
- [ ] 7. 撰写执行摘要
- [ ] 8. 完成各风险章节分析
- [ ] 9. 制作风险评估矩阵
- [ ] 10. 提出建议与对策
- [ ] 11. 标注数据来源
- [ ] 12. 审核报告完整性和准确性
```

## 注意事项

1. **数据时效性**：优先使用最近1年内的数据
2. **多源验证**：重要信息至少2个来源交叉验证
3. **合规边界**：不涉及非公开信息，不做投资建议
4. **免责声明**：报告末尾需加入适当的免责声明
